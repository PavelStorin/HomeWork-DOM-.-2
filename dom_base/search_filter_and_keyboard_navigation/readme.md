# DOM Practice — Search Filter with Keyboard Navigation

Небольшая задача для практики **обработки событий клавиатуры, фильтрации элементов и управления состоянием списка**.

Цель — реализовать **поиск по списку с клавиатурной навигацией**.

Это упражнение тренирует:

- работу с `input` событиями
- фильтрацию DOM-элементов
- обработку клавиатуры (`keydown`)
- управление активным элементом списка

---

# 📋 Задача

На странице есть **поле поиска** и **список элементов**.

Необходимо реализовать JavaScript-логику, которая позволит:

- фильтровать список при вводе текста
- очищать поиск клавишей **Escape**
- фокусировать поле поиска комбинацией **Ctrl + K**
- перемещаться по элементам списка с помощью **ArrowUp / ArrowDown**

---

# 🧱 Структура страницы

Главные элементы:

| Элемент   | Назначение       |
| --------- | ---------------- |
| `#search` | поле поиска      |
| `#list`   | список элементов |
| `li`      | элемент списка   |

HTML страницы:

```html
<div class="search-box">
  <input id="search" placeholder="Search..." />

  <ul id="list">
    <li>JavaScript</li>
    <li>React</li>
    <li>Angular</li>
    <li>Vue</li>
    <li>Svelte</li>
  </ul>
</div>
```

---

# 🎨 Как работают стили

Элемент списка может иметь класс `active`.

```css
li.active {
  background: #1677ff;
  color: white;
}
```

Этот класс используется для **выделения текущего выбранного элемента** при навигации стрелками.

JavaScript должен **добавлять и удалять класс `active`**.

---

# ✅ Требования

Необходимо реализовать следующие функции.

---

### Фильтрация списка

При вводе текста в поле поиска:

- элементы списка должны фильтроваться
- отображаются только элементы, содержащие введённый текст

Подсказка: можно использовать `textContent`.

---

### Очистка поиска

При нажатии **Escape**:

- поле поиска очищается
- список возвращается к исходному состоянию

---

### Фокус на поле поиска

При нажатии **Ctrl + K**:

- курсор должен автоматически перейти в поле поиска

Подсказка: используйте метод `focus()`.

---

### Навигация по списку

Клавиши:

- **ArrowDown** — перейти к следующему элементу
- **ArrowUp** — перейти к предыдущему элементу

Выбранный элемент должен получать класс:

```
active
```
